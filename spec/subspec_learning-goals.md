### Teil von PrepSmart · Stand: 13. Juni 2026 · wiederverwendbar (Multi-User)

> **Zweck:** Definiert das Wohin. Pro Nutzer eine Instanz; speist Test-Bewertung und das tägliche Ziel-Feedback (Haupt-Spec §6, §7.1).

## Generisches Schema (Felder)
| Feld | Beschreibung |
|---|---|
| `user_id` / `tenant_id` | Identität & Mandant |
| `north_star` | übergeordnetes Ziel |
| `sub_goals` | Teilziele (fachlich + Führung) |
| `success_criteria` | woran Erfolg gemessen wird |
| `horizon` | Zeithorizont / Phasen |

## Konkrete Instanz — aktueller Nutzer
- **north_star:** Bester Lead Context Engineer für Conversational Agents im Versicherungsbereich.
- **sub_goals:**
  - Fachlich: Azure AI Foundry Evaluation beherrschen (offizielle 7er-Taxonomie, 36 Evaluatoren), danach AWS vergleichend.
  - Führung: Literal Leadership (Klarheit, Transparenz, Authentizität, Timing).
  - Praxis: im Projektalltag Übersicht behalten, Gespräche ins Ziel lenken, Team auf Kurs halten.
- **success_criteria:** wöchentliches Kompetenzprofil (Score je offizieller Evaluator-Gruppe) + Führungs-Score; Selbst- vs. Fremdbild konvergiert.
- **horizon:** 7-Tage-Kickstart (nur Microsoft) → Dauerbetrieb (AWS vergleichend, ohne Ende).
